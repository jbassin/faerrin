# Better CLI — commander.js Migration Plan

## Overview

Replace the hand-rolled argv dispatcher and per-handler string-array parsers with
[commander.js](https://github.com/tj/commander.js/), giving every command proper
`--help` output, typed option parsing, and a consistent registration model.

## Current State Analysis

- **`index.ts`** — slices `process.argv`, looks up handler by name in a flat `handlers` map, calls
  `handler(argv: string[])`. Global help lists command names only (no descriptions).
- **`src/cli/index.ts`** — exports `CliHandler = (argv: string[]) => Promise<void> | void` and the
  `handlers` record.
- **13 commands** — each handler does its own inline flag parsing (`argv.includes`, `argv[0]`,
  index-walking loops). No per-command `--help`.
- **`transcripts`** — has an internal sub-command switch (`argv[0]` = `list | status | reset`).
- **`process` / `process-all`** — two separate top-level commands sharing a manual `parseProcessArgs`
  function (`process.ts:117`).
- **9 test files** — all call handler functions with raw `string[]` (e.g.
  `segment(['2025-8-28'], opts)`).
- **No `commander` dependency** — not in `package.json`.

## Desired End State

- `bun run <command> --help` prints a description and documented flags for every command.
- Every handler file exports:
  - A typed action function: `export async function segment(name, flags, opts?)` — tested directly.
  - `export function register(program: Command)` — wires the commander command.
- `src/cli/index.ts` exports `registerAll(program: Command)` instead of a `handlers` map.
- `index.ts` creates a commander `Program`, calls `registerAll`, calls `program.parseAsync`.
- `process` and `process-all` are unified as `process [name]` with `--all`.
- `package.json` no longer has a `process-all` script.
- `bun test` and `bun run typecheck` pass.

### Key Discoveries

- `process.ts:117` — `parseProcessArgs` is unit-tested directly in `process.test.ts:225`; those
  tests must be replaced by tests of the new typed action function.
- `process.ts` exports `processOneTranscript`, `LedgerMutex`, `getTargets` — these stay exported
  and tested; only `processTranscript`, `processAll`, and `parseProcessArgs` are removed.
- `index-wiki` uses `--no-llm`; commander maps `--no-X` to `opts.X === false`, so the typed flag
  becomes `{ llm?: boolean }` where `false` means "skip LLM".
- `cost-report` has no test file; `hello` and `index-wiki` have no test files — signature changes
  only, no test updates needed.
- `transcripts.test.ts` calls `transcripts(['list'], opts)` etc. — these change in Phase 2.

## What We're NOT Doing

- Changing any domain/pipeline logic under `src/transcript/`, `src/reconcile/`, `src/gitlab/`,
  `src/wiki/`.
- Adding new commands or flags beyond what already exists.
- Changing `package.json` scripts for individual commands (they still route through
  `bun index.ts <command>`).
- Adding global middleware or plugin infrastructure to the commander program.

## Implementation Approach

Two phases, each leaving `bun test` and `bun run typecheck` green.

**Phase 1** migrates the 11 simple (single-level) handlers, rewrites the root entry point, and
inserts thin pass-through shims for `transcripts` and `process`/`process-all` so the root is
functional while those two are still on the old interface.

**Phase 2** converts the two complex handlers, removes shims, consolidates `process`/`process-all`,
and updates the remaining test files.

## Phase 1: Install + Simple Handlers + Root Wiring

### Overview

Install commander, convert all single-level handlers to the new pattern (typed action function +
`register`), update their tests, and rewrite the root with pass-through shims covering the two
not-yet-converted handlers.

### Changes Required

#### 1. Install commander

```sh
bun add commander
```

#### 2. Handler pattern (apply to each simple handler below)

Each handler file changes from:

```ts
export async function segment(argv: string[], opts: SegmentCliOptions = {}): Promise<void> {
  const all  = argv[0] === '--all';
  const name = all ? undefined : argv[0];
  // ...
}
```

To:

```ts
import { Command } from 'commander';

export async function segment(
  name: string | undefined,
  flags: { all?: boolean },
  opts: SegmentCliOptions = {},
): Promise<void> {
  // same body — replace all argv-parsing locals with typed flags
  // e.g. replace: const all = argv[0] === '--all';  name = argv[0]
  //      with:    flags.all  and  name  (already typed parameters)
}

export function register(program: Command): void {
  program
    .command('segment [name]')
    .description('Segment a transcript into labeled scenes')
    .option('--all', 'process all unsegmented transcripts')
    .action((name: string | undefined, opts: { all?: boolean }) => segment(name, opts));
}
```

#### 4. Rewrite `src/cli/index.ts`

Remove the `CliHandler` type and `handlers` map. Export `registerAll`:

```ts
import { Command } from 'commander';
import { register as registerHello }      from './hello';
import { register as registerCostReport } from './cost-report';
import { register as registerIndexWiki }  from './index-wiki';
import { register as registerSegment }    from './segment';
import { register as registerExtract }    from './extract';
import { register as registerResolve }    from './resolve';
import { register as registerMatch }      from './match';
import { register as registerPropose }    from './propose';
import { register as registerSubmit }     from './submit';
import { register as registerRespond }    from './respond';

// Phase 1 shims — replaced in Phase 2
import { transcripts }                          from './transcripts';
import { processTranscript, processAll }        from './process';

export function registerAll(program: Command): void {
  registerHello(program);
  registerCostReport(program);
  registerIndexWiki(program);
  registerSegment(program);
  registerExtract(program);
  registerResolve(program);
  registerMatch(program);
  registerPropose(program);
  registerSubmit(program);
  registerRespond(program);

  // Temporary pass-through shims until Phase 2
  program.command('transcripts [args...]')
    .description('Manage the transcript ledger')
    .allowUnknownOption()
    .passThroughOptions()
    .action((_first, _opts, cmd) => transcripts(cmd.args));

  program.command('process [args...]')
    .description('Run the full pipeline for one transcript')
    .allowUnknownOption()
    .passThroughOptions()
    .action((_first, _opts, cmd) => processTranscript(cmd.args));

  program.command('process-all [args...]')
    .description('Run the full pipeline for all transcripts')
    .allowUnknownOption()
    .passThroughOptions()
    .action((_first, _opts, cmd) => processAll(cmd.args));
}
```

#### 2. Consolidate `src/cli/process.ts`

Replace `processTranscript(argv)`, `processAll(argv)`, and `parseProcessArgs` with a single
typed action function:

```ts
export async function process(
  name: string | undefined,
  flags: {
    all?:        boolean;
    dryRun?:     boolean;
    force?:      string;
    stopBefore?: string;
    concurrency?: number;
  },
  opts: ProcessCliOptions = {},
): Promise<void> {
  // Validation (mirrors current processTranscript / processAll guard logic):
  if (!name && !flags.all) {
    console.error('usage: bun run process <name> [--dry-run] [--force <stage>] [--stop-before <stage>]');
    console.error('       bun run process --all [--dry-run] [--stop-before <stage>] [--concurrency <n>]');
    process.exit(1);
  }
  if (name && flags.all) {
    console.error('cannot combine a transcript name with --all');
    process.exit(1);
  }
  if (!flags.all && flags.concurrency && flags.concurrency > 1) {
    console.error('--concurrency is only valid with --all');
    process.exit(1);
  }
  if (flags.all && flags.force) {
    console.error('--force is not allowed with --all');
    process.exit(1);
  }
  // ... rest of logic (currently spread across processTranscript / processAll bodies)
}

export function register(program: Command): void {
  program
    .command('process [name]')
    .description('Run the full pipeline for one transcript or all')
    .option('--all',                'process all eligible transcripts')
    .option('--dry-run',            'skip actual GitLab submission')
    .option('--force <stage>',      `re-run from this stage (${PIPELINE_STAGES.join(', ')})`)
    .option('--stop-before <stage>','halt pipeline before this stage')
    .option('--concurrency <n>',    'parallel workers (--all only)', (v) => parseInt(v, 10), 1)
    .action((name: string | undefined, opts) => process(name, opts));
}
```

Keep exported: `processOneTranscript`, `LedgerMutex`, `getTargets` — tests reference these.
Remove: `processTranscript`, `processAll`, `parseProcessArgs`, `ProcessArgs` type.

Update `process.test.ts`:

- Remove the `parseProcessArgs` describe block (`process.test.ts:223-289`). The validation
  behaviour is now covered by the guard checks in the unified `process` action.
- Replace `processTranscript([...], opts)` calls with `process(name, flags, opts)`.
- Replace `processAll([...], opts)` calls with `process(undefined, { all: true, ...flags }, opts)`.

Concrete substitution examples:

| Old | New |

| `processTranscript(['--concurrency', '2', '000'], opts)` | `process('000', { concurrency: 2 }, opts)` |
| `processTranscript(['000', '--force', 'propose', '--stop-before', 'propose'], opts)` | `process('000', { force: 'propose', stopBefore: 'propose' }, opts)` |
| `processAll([], opts)` | `process(undefined, { all: true }, opts)` |
| `processAll(['--stop-before', 'segment'], opts)` | `process(undefined, { all: true, stopBefore: 'segment' }, opts)` |
| `processAll(['--concurrency', '2'], opts)` | `process(undefined, { all: true, concurrency: 2 }, opts)` |
| `processAll(['--force', 'propose'], opts)` | `process(undefined, { all: true, force: 'propose' }, opts)` (expects exit 1) |

## Testing Strategy

### Unit Tests
- Each action function is tested by calling it directly with typed arguments — no commander
  machinery in tests.
- Tests pass explicit `opts` (injected dirs, models, fake `completeFn`) to keep them hermetic.
- The `parseProcessArgs` unit tests are removed; equivalent coverage comes from `process` action
  tests that pass typed flags and assert guard-clause behaviour via `withMockedExit`.

### Manual Testing
1. `bun index.ts --help` — check all 12 commands listed with descriptions.
2. `bun run segment --help` — verify argument and option descriptions present.
3. `bun run process --help` — verify all flags present and documented.
4. `bun run transcripts --help` — verify subcommands listed.
5. `bun run process` (no args) — expect clean usage error, not a stack trace.
